# AutoOnCall 文档入口

这个目录里的文档分为三类：面试入口、校招改进计划、深入材料。第一次了解项目时，不建议从所有技术长文开始读，先按下面顺序建立主线。

## 必读三件套

1. [README](../README.md)
   项目第一入口，先看项目叙事、10 分钟演示路径、核心链路和质量验证。

2. [从零上手与面试讲解](AutoOnCall项目从零上手与面试讲解.md)
   面试讲法、代码地图、固定演示脚本、常见追问和简历写法。

3. [校招竞争力改进计划](AutoOnCall校招竞争力改进计划.md)
   两个月冲刺取舍、P0/P1/P2 改进路线、任务拆分和验收标准。

读完这三份，就应该能回答：项目解决什么问题、主链路怎么跑、两个月内优先改什么、哪些能力不能夸大、交付前如何验证。

## 深入阅读

- [项目分析书](AutoOnCall项目分析书.md)：系统设计、价值拆解、能力边界和校招评分。
- [校招生项目评估与改进建议](AutoOnCall校招生项目评估与改进建议.md)：阶段性评分、短板和后续优化顺序。
- [校招竞争力对标与改进清单](校招竞争力对标与改进清单.md)：对标 HolmesGPT、RCA Agent、MCP servers 等项目后的原始调研清单。
